## Estrutura da Entidade

A entidade **DadosPessoaFisica** representa as informações pessoais de uma pessoa física no sistema. Ela contém informações como nome, sobrenome, data de nascimento, sexo, grau de instrução, CAF e data de validade do CAF.

### Anotações

- `@Getter` e `@Setter`: Geram automaticamente os métodos getters e setters para os campos da entidade.
- `@Entity`: Indica que a classe é uma entidade JPA.
- `@Table(name = "dados_pessoa_fisica")`: Especifica o nome da tabela no banco de dados.
- `@Audited` e `@AuditTable("dados_pessoa_fisica_aud")`: Habilitam a auditoria para a entidade e especificam o nome da tabela de auditoria.
- `@NoArgsConstructor`: Gera um construtor sem argumentos.
- `@JsonDeserialize` e `@JsonSerialize`: Especificam os deserializadores e serializadores para os campos de data.

### Campos

- `id`: Identificador único dos dados pessoais.
- `nome`: Nome da pessoa.
- `sobrenome`: Sobrenome da pessoa.
- `dataNascimento`: Data de nascimento da pessoa.
- `sexo`: Sexo da pessoa (enum `Sexo`).
- `grauInstrucao`: Grau de instrução da pessoa (enum `GrauInstrucao`).
- `caf`: Código CAF da pessoa.
- `dataValidadeCaf`: Data de validade do código CAF.

### Construtor

```java
public DadosPessoaFisica(DadosPessoaFisicaRequestDto request) {
  nome = request.getNome();
  sobrenome = request.getSobrenome();
  dataNascimento = request.getDataNascimento();
  sexo = request.getSexo();
  grauInstrucao = request.getGrauInstrucao();
  caf = request.getCaf();
  dataValidadeCaf = request.getDataValidadeCaf();
}
```

Este construtor inicializa um objeto `DadosPessoaFisica` com base nos dados fornecidos em um `DadosPessoaFisicaRequestDto`.
