## Visão Geral

A estrutura do projeto é organizada de forma a garantir a separação de responsabilidades e a modularidade do código. Abaixo estão descritos os principais componentes da aplicação e suas responsabilidades.

## Componentes Principais

### Controller

Os **controladores** são responsáveis por gerenciar as requisições HTTP e fornecer uma interface RESTful para as funcionalidades do sistema. Eles recebem as solicitações dos clientes, processam as informações necessárias e retornam as respostas apropriadas. Os controladores são uma parte essencial da arquitetura de uma aplicação baseada em serviços, pois garantem a separação clara entre a camada de apresentação e a lógica de negócios.

### Service

Os **serviços** são responsáveis por implementar a lógica de negócios e fornecer funcionalidades específicas para os controladores. Eles encapsulam a lógica de processamento de dados e interagem com o repositório de dados para realizar operações de leitura e gravação. Os serviços são uma parte essencial da arquitetura de uma aplicação baseada em serviços, pois garantem a separação clara entre a camada de controle e a camada de persistência.

<Outlet path="/project-structure/service" />

### Repository

Os **repositórios** são responsáveis por interagir com o banco de dados. Eles fornecem métodos para realizar operações de leitura e gravação de dados. Os repositórios são uma parte essencial da arquitetura de uma aplicação baseada em serviços, pois garantem a separação clara entre a camada de persistência e a lógica de negócios.

### Model

Os **modelos** representam as entidades do domínio da aplicação. Eles são usados para mapear os dados do banco de dados para objetos Java. Os modelos são uma parte essencial da arquitetura de uma aplicação baseada em serviços, pois garantem a consistência dos dados e a integridade do domínio.

### DTO

Os **DTOs (Data Transfer Objects)** são usados para transferir dados entre as camadas da aplicação. Eles são usados para encapsular os dados que são enviados e recebidos pelas APIs. Os DTOs são uma parte essencial da arquitetura de uma aplicação baseada em serviços, pois garantem a separação clara entre as camadas e a segurança dos dados.

### Util

As **classes utilitárias** fornecem métodos auxiliares que são usados em várias partes da aplicação. Elas são usadas para encapsular lógica comum e reutilizável. As classes utilitárias são uma parte essencial da arquitetura de uma aplicação baseada em serviços, pois garantem a reutilização de código e a manutenção da aplicação.

### Config

As **classes de configuração** são responsáveis por configurar os componentes da aplicação. Elas são usadas para definir beans, configurar propriedades e inicializar componentes. As classes de configuração são uma parte essencial da arquitetura de uma aplicação baseada em serviços, pois garantem a flexibilidade e a extensibilidade da aplicação.
